A project in Bluplai is the working record for a specific engagement, initiative, or program you're running with a customer account. While an account holds the overall relationship, a project is where you track the active work — the success plan, milestones, deliverables, tasks, and progress. Every project belongs to exactly one customer account.

## Key fields

When you create or edit a project, you work with these fields:

| Field | Options | Description |
|---|---|---|
| **Name** | Free text | A clear name for the engagement. Required. |
| **Description** | Free text | Optional context about the project's purpose. |
| **Account** | Select | The customer account this project belongs to. Required. |
| **Lifecycle stage** | Discover → Extend | Where this project sits in the customer lifecycle. |
| **Status** | Active, On hold, Completed, Archived | The current operational state of the project. |
| **Priority** | Low, Medium, High, Critical | How urgently the project needs attention. |
| **Risk level** | Low, Medium, High | The current risk level for the engagement. |

## Project status

Status reflects where the project stands operationally. Use it to communicate the state of the work to your team.

<AccordionGroup>
  <Accordion title="Active">
    The project is underway. Work is progressing and the project is being actively managed.
  </Accordion>
  <Accordion title="On hold">
    The project has been paused. Use this when work has stopped temporarily — for example, waiting on a customer decision or a resource becoming available.
  </Accordion>
  <Accordion title="Completed">
    All planned work is done. The project has achieved its objectives.
  </Accordion>
  <Accordion title="Archived">
    The project is no longer active and has been moved out of the working view. Archive projects you no longer need to reference regularly.
  </Accordion>
</AccordionGroup>

The typical flow is: **Active → On hold or Completed → Archived**.

## How to create a project

<Steps>
  <Step title="Go to Projects">
    Click **Projects** in the left sidebar to open the projects list.
  </Step>
  <Step title="Click the create button">
    Click the **+** button or **New Project** button to open the creation dialog.
  </Step>
  <Step title="Select the customer account">
    Choose the account this project belongs to from the dropdown. This is required.
  </Step>
  <Step title="Name the project">
    Give the project a clear, descriptive name. This is the only other required field.
  </Step>
  <Step title="Set the lifecycle stage">
    Choose the stage that best reflects where this engagement currently sits. You can change this as the engagement progresses.
  </Step>
  <Step title="Set optional fields and save">
    Optionally add a description, priority, and risk level. Click **Create Project** to save.
  </Step>
</Steps>

## What a project contains

Once a project is created, you can fill it out using the project's modules. From the project detail view, you can access:

<CardGroup cols={3}>
  <Card title="Digital Assessment" icon="chart-bar">
    Capture the customer's goals, workflows, technology stack, and current-state analysis.
  </Card>
  <Card title="Success Plan (CSP)" icon="clipboard-check">
    Build a customer success plan with initiatives, actions, milestones, and a timeline.
  </Card>
  <Card title="Value Analysis" icon="dollar-sign">
    Track ROI calculations, benefits, assumptions, and key business impacts.
  </Card>
</CardGroup>

In addition to these modules, a project also tracks:

- **Initiatives** — the major work streams within the project.
- **Milestones** — key dates and checkpoints.
- **Deliverables** — outputs that need to be produced and approved.
- **Tasks** — individual actions assigned to team members.
- **Stakeholders** — the people from the customer account who are engaged on this project (linked from the account level).

## Filtering and searching projects

On the Projects list page you can:

- **Search** by project name to find a specific engagement.
  - **Filter by lifecycle stage** using the stage filter controls at the top of the page.
- **Filter by status** to focus on active or on-hold work.

<Tip>
  Use the lifecycle stage filter to focus your team's attention on a specific cohort — for example, all projects currently in the Onboard stage.
</Tip>

## Editing and deleting projects

To edit a project's name, status, priority, lifecycle stage, or risk level, open the project list and click the edit icon on the project card, or open the project detail and use the edit option from the header.

<Note>
  Deleting a project removes it and all its associated data — initiatives, milestones, tasks, deliverables, and CSP content. If you want to preserve the history, set the status to **Archived** instead of deleting.
</Note>
